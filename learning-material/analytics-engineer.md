# Analytics Engineer Learning Map

## Core Outcomes

The learner should be able to transform raw data into trusted analytical
models and explain metric logic.

## Topics

- Dimensional modeling
- Fact and dimension tables
- Grain
- dbt-style model layering
- Source freshness
- Data tests
- Documentation
- Metric definitions
- Lineage
- Stakeholder trust

## Project Evidence

- Clear model layers
- Fact table with documented grain
- Dimensions that support analysis
- Tests for uniqueness, nulls, and accepted values
- Metric definitions with caveats

## Interview Practice

- Explain the grain of a fact table.
- Explain how you prevent metric confusion.
- Explain how you test models.
- Explain a tradeoff in the model design.
