# Data Engineer Learning Map

## Core Outcomes

The learner should be able to build and explain a reliable data pipeline.

## Topics

- Reproducible environments with `uv`
- Batch ingestion
- File formats and schemas
- SQL transformation
- Data validation
- Idempotency
- Incremental loading
- Partitioning basics
- Pipeline observability
- Testing and failure recovery

## Project Evidence

- Clear source-to-output workflow
- Repeatable run command
- Validation checks
- Handling for duplicates or late data
- Explanation of failure modes

## Interview Practice

- Explain idempotency.
- Explain how you detect bad data.
- Explain how you would scale or schedule the pipeline.
- Explain what you would monitor in production.
